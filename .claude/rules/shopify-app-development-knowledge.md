# Shopify App Development Knowledge & Rules

Rules and patterns for agents implementing SmartRec and any features on this Shopify embedded app.

---

## Architecture Overview

This is a **Shopify embedded app** — it runs inside the Shopify Admin iframe, NOT as a standalone website.

```
Storefront (merchant theme)
  └─ Theme App Extension (app embed block)
       └─ fetch(/apps/smartrec/*) ──→ App Proxy ──→ App Server (proxy.$.tsx)

Shopify Admin
  └─ Embedded iframe ──→ App Server (app/* routes, Polaris web components)
```

---

## Critical Shopify Patterns (MUST follow)

### Authentication — 3 contexts, use the right one

| Context | When | Import |
|---------|------|--------|
| `authenticate.admin(request)` | Admin routes (`app/*`) — loaders & actions | Protected, has session + Admin API access |
| `authenticate.webhook(request)` | Webhook handlers (`webhooks.app.*`) | Validates HMAC, returns shop + topic + payload |
| `authenticate.public.appProxy(request)` | Proxy routes (`proxy.*`) — storefront requests | Validates Shopify signature, may or may not have session |

**Rules:**
- NEVER use `authenticate.admin` in proxy routes (storefront has no admin session)
- NEVER use `authenticate.public.appProxy` in admin routes
- ALWAYS authenticate before any other logic in the handler
- App proxy returns `session: undefined` if store hasn't installed the app — handle this

### Navigation — embedded app rules

- Use `Link` from `react-router` or Polaris — NEVER use raw `<a>` tags
- Use `redirect` from `authenticate.admin` result — NOT from `react-router`
- All `/app/*` routes are inside the authenticated layout (`app.tsx`)

### Polaris Web Components (NOT React components)

This app uses **Polaris web components** (`s-page`, `s-button`, `s-card`, etc.), NOT `@shopify/polaris` React components.

```tsx
// CORRECT — web components
<s-page title="Settings">
  <s-card>
    <s-button variant="primary" onClick={handleSave}>Save</s-button>
  </s-card>
</s-page>

// WRONG — React Polaris (do NOT use)
import { Page, Card, Button } from "@shopify/polaris";
```

- Types from `@shopify/polaris-types` (in tsconfig `types` array)
- `shopify` is a readonly global (App Bridge) — use `shopify.toast.show()` etc.
- Always use `useAppBridge()` hook when you need App Bridge APIs in components

### GraphQL — Admin API only via authenticated session

```tsx
export const loader = async ({ request }) => {
  const { admin } = await authenticate.admin(request);
  const response = await admin.graphql(`
    #graphql
    query { shop { name } }
  `);
  const data = await response.json();
  return data;
};
```

- Always prefix queries with `#graphql` comment for syntax highlighting
- Types generated via `npm run graphql-codegen` to `app/types/`
- Current API version: `October25` (set in `app/shopify.server.ts`)

### Webhooks — declared in TOML, handled in routes

- Webhook topics are declared in `shopify.app.toml` `[[webhooks.subscriptions]]`
- Handler routes follow naming: `webhooks.app.<topic>.tsx` (dots become dashes)
- Example: `orders/create` → `webhooks.app.orders-create.tsx`
- Webhook handlers export `action` (POST), never `loader`
- Always destructure `{ shop, topic, payload }` from `authenticate.webhook()`

### App Proxy — storefront-to-app communication

- Configured in `shopify.app.toml` `[app_proxy]` section
- Storefront URL: `https://<shop>/apps/smartrec/*`
- Routes to: `app/routes/proxy.$.tsx` (catch-all)
- Shopify adds query params: `shop`, `logged_in_customer_id`, `path_prefix`, `timestamp`, `signature`
- App proxies do NOT support cookies — use query params or request body for state
- HMAC signature verified by `authenticate.public.appProxy()`
- Responses can be JSON or Liquid (use `liquid()` helper for Liquid responses)

## Database (Prisma + SQLite)

- Schema: `prisma/schema.prisma`
- Migrations: `npx prisma migrate dev --name <name>` (dev) or `prisma migrate deploy` (prod)
- Client: import from `app/db.server.ts`
- Session model managed by `@shopify/shopify-app-session-storage-prisma` — do NOT modify
- After schema changes: run `npx prisma generate` to update client types

## Routing (React Router v7 flat file-system)

| File Pattern | URL | Purpose |
|-------------|-----|---------|
| `app.tsx` | — | Authenticated layout wrapper |
| `app._index.tsx` | `/app` | Dashboard |
| `app.<name>.tsx` | `/app/<name>` | Admin pages |
| `auth.$.tsx` | `/auth/*` | OAuth callback catch-all |
| `webhooks.app.<topic>.tsx` | `/webhooks/app/<topic>` | Webhook handlers |
| `proxy.$.tsx` | `/proxy/*` | App proxy catch-all |

- Routes auto-discovered by `@react-router/fs-routes` (`flatRoutes()` in `app/routes.ts`)
- Route types generated by `react-router typegen` to `.react-router/types/`

## SmartRec-Specific Decisions (from plan validation)

- **Scopes:** Keep all existing scopes (write_products, etc.) — app may have other features
- **Proxy subpath:** `smartrec` (NOT the app name `plug-play`)
- **Storage:** SQLite + in-memory Node.js Map cache for hot data. No Redis for MVP.
- **Intent score:** Client-side calculation only. Server trusts client score.
- **AI assistant code:** `app/ai-assistant/` is legacy — leave as-is, don't fix imports, don't delete
- **Proxy routing:** Single catch-all `proxy.$.tsx` with switch dispatcher (not separate route files)

## Common Mistakes to Avoid

1. **Don't import from `@remix-run/*`** — this is React Router v7, use `react-router` imports
2. **Don't use `json()` from react-router** — use `Response.json()` directly
3. **Don't create standalone pages** — all admin UI must be embedded (under `app.*` routes)
4. **Don't register webhooks in code** — they're declared in `shopify.app.toml`
5. **Don't use `fetch()` for Admin API** — use `admin.graphql()` from authenticated context
6. **Don't modify the Session prisma model** — managed by Shopify session storage package
7. **Don't add CORS headers to proxy routes** — app proxy is same-origin (Shopify handles this)
8. **Don't use `process.env` for shop-specific data** — get shop from auth context or query params
